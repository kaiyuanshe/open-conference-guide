# 开放会务指南

[![Join the chat at https://gitter.im/kaiyuanshe/open-conference-guide](https://badges.gitter.im/kaiyuanshe/open-conference-guide.svg)](https://gitter.im/kaiyuanshe/open-conference-guide?utm_source=badge&utm_medium=badge&utm_campaign=pr-badge&utm_content=badge)

## 前言

这篇文章的读者为有志于自发组织一场开源社区的会议，尤其是主要利用业余时间和志愿者资源的开源爱好者和贡献者们。本手册是为中国开源年会 COSCon 而写，但是也适用于其他的一些社区非营利会议。

首先，感谢你做出决定：要投入时间和精力去办一场为期 2 天的 COSCon，这不是一个容易的决定。一旦决定要作这件事情，就意味着你要跟十几位 COSCon 的核心组委会小伙伴们花 4 个月的时间每周至少一个小时的电话会议；在会议召开前的三天惴惴不安，面对各种不完美、难以完成的任务；也意味着你将在你最擅长的领域去贡献，竭尽所能。

组织这样一场会议，也会获得你难以想象的回报。这种回报跟金钱无关，而是你能体会到马斯洛人类需求理论里自我实现那一层的快乐。在到达目标之前会经历各种困难和挫折，而在经历完这些历练之后，你回过头再去看，会看到自己的成长；而这基于痛苦之后的成长，将会是我们未来开源和个人事业之路上的宝库。

## 指南结构

一场大会，千头万绪。要想涵盖所有的方方面面，非常困难。本指南先将会务工作整理为三个部分：基础认知、筹备执行、现场与收尾。每一章都对应一个明确文件，方便后续逐章补写、审阅和协作。

## 作为 Agent Skill 使用

本仓库根目录提供了 [`SKILL.md`](SKILL.md)。把本仓库作为 skill 安装到支持 `SKILL.md` 的 agent 后，用户可以直接让 agent 基于这套知识库协助制定会议计划、生成模板、检查风险和推进执行。

适合的提问方式：

1. “帮我为一个 100 人的开源 Meetup 制定 6 周筹备计划。”
1. “基于这个 skill，帮我写一份 CFP 文案和讲师邀请信。”
1. “帮我检查这份会议计划还有哪些风险。”
1. “为我的社区会议生成现场 Runbook、志愿者分工表和物料清单。”

安装方式取决于你的 agent 环境。通常可以将整个仓库目录复制或克隆到 agent 的 skills 目录中，例如命名为 `open-conference-guide`。安装后，agent 会读取 `SKILL.md`，并按需引用本仓库中的章节和 `templates/` 模板。
