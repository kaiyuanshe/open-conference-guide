# 开放会务指南

[![Join the chat at https://gitter.im/kaiyuanshe/open-conference-guide](https://badges.gitter.im/kaiyuanshe/open-conference-guide.svg)](https://gitter.im/kaiyuanshe/open-conference-guide?utm_source=badge&utm_medium=badge&utm_campaign=pr-badge&utm_content=badge)

## 前言

这篇文章的读者为有志于自发组织一场开源社区的会议，尤其是主要利用业余时间和志愿者资源的开源爱好者和贡献者们。 本手册是为中国开源年会COSCon而写，但是也适用于其他的一些社区非盈利会议。 

首先，感谢你做出决定：要投入时间和精力去办一场为期2天的COSCon，这不是一个容易的决定，一旦决定要作这件事情， 就意味着你要跟十几位COSCon的核心组委会小伙伴们花4个月的时间每周至少一个小时的电话会议；在会议召开前的三天惴惴不安，各种不完美，难以完成的任务；也意味着你将在你最擅长的领域去贡献，竭尽所能。 组织这样一场会议，也会获得你难以想象的回报，这种回报跟金钱无关，而是你能体会到马斯洛人类需求理论里自我实现的那一层的快乐。 在到达目标之前会经历各种困难和挫折， 而在经历完这些历练之后， 你回过头再去看，会看到自己的成长，而这基于痛苦之后的成长，将会是我们未来开源和个人事业之路上的宝库。 

## 指南的结构

一场大会，千头万绪。要想涵盖所有的方方面面，非常困难。本文试图首先梳理出这样的一个概念结构，然后一一叙述。也欢迎所有的朋友，补充和完善这一结构。
